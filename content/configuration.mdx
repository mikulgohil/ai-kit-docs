# Configuration

AI Kit generates configuration automatically from your project's tech stack. Here's how to understand and customize what's generated.

## ai-kit.config.json

This file tracks what AI Kit generated and the scan results. It's created by `ai-kit init` and updated by `ai-kit update`.

```json
{
  "version": "1.0.0",
  "scanResult": {
    "framework": "nextjs",
    "nextjsVersion": "15.1.0",
    "routerType": "app",
    "cms": "sitecore-xmc",
    "sitecorejssVersion": "22.3.0",
    "styling": ["tailwind"],
    "tailwindVersion": "4.0.0",
    "typescript": true,
    "typescriptStrict": true,
    "monorepo": false,
    "figma": {
      "detected": true,
      "figmaMcp": true,
      "figmaCodeCli": false,
      "designTokens": true,
      "tokenFormat": "tailwind-v4",
      "visualTests": false
    },
    "packageManager": "npm",
    "projectName": "my-project",
    "scripts": { "dev": "next dev", "build": "next build" }
  },
  "generatedAt": "2026-03-17T10:00:00.000Z",
  "templates": ["CLAUDE.md", ".cursorrules"],
  "commands": ["prompt-help", "review", "fix-bug"],
  "guides": ["getting-started", "prompt-playbook"]
}
```

### Fields

| Field | Description |
|---|---|
| `version` | AI Kit version that generated this config |
| `scanResult` | Complete tech stack detection results |
| `generatedAt` | ISO timestamp of last generation |
| `templates` | Which template files were generated |
| `commands` | Which slash commands were copied |
| `guides` | Which guides were copied |

## Customizing Generated Rules

### Adding Custom Rules

The generated `CLAUDE.md` and `.cursorrules` files are wrapped in markers:

```markdown
<!-- AI-KIT:START -->
[auto-generated content]
<!-- AI-KIT:END -->
```

**Add your custom rules outside the markers** â€” above or below:

```markdown
# My Custom Rules

These rules are specific to this project and won't be overwritten by ai-kit update.

- Always use the `useTranslation` hook for text strings
- API calls must go through `src/lib/api-client.ts`
- Feature flags are managed via LaunchDarkly â€” use `useFeatureFlag` hook

<!-- AI-KIT:START -->
[auto-generated content]
<!-- AI-KIT:END -->

# Additional Notes

- The legacy API is at /api/v1 â€” don't add new endpoints there
- Contact @sarah for design system questions
```

When you run `ai-kit update`, only the content between the markers is refreshed. Everything else stays untouched.

### Editing Generated Content

You can edit content **inside** the markers, but those edits will be lost on `ai-kit update`. If you need to modify a generated rule:

1. Add your override **outside** the markers
2. Or modify the rule inside markers but remember it resets on update

### Per-File Rules (Cursor .mdc)

## Template Fragments

AI Kit selects template fragments based on your scan. Here's the logic:

```
base              â†’ Always included
nextjs-app-router â†’ framework === 'nextjs' AND routerType is 'app' or 'hybrid'
nextjs-pages-router â†’ framework === 'nextjs' AND routerType is 'pages' or 'hybrid'
sitecore-xmc      â†’ cms !== 'none'
tailwind          â†’ 'tailwind' in styling array
typescript        â†’ typescript === true
monorepo          â†’ monorepo === true
figma             â†’ figma.detected === true
```

Templates live in the AI Kit package at `templates/claude-md/` and `templates/cursorrules/`.

## Git Integration

### What to commit

```gitignore
# Commit these â€” they're project configuration

.cursorrules
.cursor/rules/
.claude/skills/
.cursor/skills/
.claude/commands/

# These are scaffolds with your content â€” definitely commit
docs/mistakes-log.md
docs/decisions-log.md
docs/time-log.md
```

### What to .gitignore

Nothing AI Kit generates needs to be gitignored. All generated files are project configuration that should be shared with the team.

### Recommended workflow

1. Run `ai-kit init` on a fresh branch
2. Review generated files
3. Commit as: `chore: add ai-kit configuration`
4. Team members pull and get the same AI context
